# 🛠 開発戦略：モック・API・Gemの多層アプローチ

本プロジェクトでは、開発効率の最大化とランニングコストの最適化を両立するため、以下の3ステップ戦略を採用しています。

---

### 🧪 STEP 1：モックデータによる「解析ロジック」の先行開発
**「外部依存を排除したロジックの構築」**
* **概要**: X APIの制限やコストに依存せず、手元のJSONファイルを用いて「ゾンビ判定」ができるRubyプログラムを先行して実装します。
* **メリット**: 開発サイクルの高速化と、境界値テスト（エッジケース）の容易な再現を可能にします。

### 💎 STEP 2：判定ロジックの「Gem化」とデカップリング
**「責務の分離による保守性の向上」**
* **概要**: Railsアプリケーションから解析ロジックを抽出し、独立したクラス・モジュールとして設計します。
* **メリット**: これを将来的に自作Gemの「核」とすることで、ビジネスロジックのカプセル化と、他プロジェクトへの高いポータビリティを実現します。

### ⚡ STEP 3：戦略的「API連携」とプロトタイピング
**「最小コストでのプロダクション検証」**
* **概要**: APIを「データの供給源（Adapter）」と定義し、最終フェーズで最小限の課金による実データ検証を行います。
* **メリット**: 予算制約がある場合でも、「API連携可能なインターフェース」をポートフォリオとして提示しつつ、デモ環境では安定したモック運用を継続できます。
