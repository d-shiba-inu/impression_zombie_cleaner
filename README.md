# 🧟Impression Zombie Cleaner

# アプリケーション概要
X（旧 Twitter）におけるインプレッションゾンビ（インプレッション（閲覧数）を増やして、収益を目的にする自動リプライロボット）を特定し、快適なタイムラインを取り戻すためのアプリケーションです。特定のポストへのリプライをスキャンし、独自アルゴリズムを用いてアカウントの属性を分析・判定します。

# アプリケーションを作成した背景
Xのリプライ欄は元ポストの信憑性やさらなる情報の獲得に有益でした。 しかし、現段階（’26 3月現在）収益目的無意味なリプライ（インプレゾンビ）が溢れ、本来の有益な情報が埋もれてしまうという状況を散見します。
このアプリケーションを通じて、情報収集を行う一般ユーザーや、自社のポストの反応を正確に分析したい企業公式アカウントが、ノイズ（ゾンビ）に邪魔されることなく快適に X を利用できる環境を提供しようと考え、開発に至りました。

# 利用方法
1. アプリケーション（URL）にアクセスします。
2. 解析したい X のポスト URL を入力フォームに貼り付けます。
3. 「スキャン開始」ボタンをクリックします。
4. リプライ一覧が表示され、各アカウントの「ゾンビスコア（判定結果）」が確認できます。
⚠️xAPI取得時に課金が発生してしまうため、現在は実際に一度取得したデータとURLを使って、解析をお試し頂いております。

# 🧹実装した機能についての説明

### 1. リアルタイム・ゾンビスキャン機能
- **特徴**: 入力された URL から非同期でリプライを取得し、即座に画面に結果を描画します。React による高速な UI レンダリングを活用しています。

### 2. インプレゾンビ自動判定機能
- **特徴**: アカウントの言語、作成日、認証バッジの有無、固定ツイートの内容などを統合的にスコアリングし、視覚的に分かりやすく判定結果を表示します。
⚠️インプレゾンビ側も日々賢くなっているため、さらなる判定の強化を測っています。

# 🗓️実装予定の機能
- **GitHub Actions による CI/CD**： デプロイ作業の完全自動化。
- **101件以上のリプライの同時解析**： 現在は一度に100件までしかリプライを取得できないため、一括解析の手段を考えています。
- **判定精度の向上**： 機械学習モデルを用いた、より高度なパターン認識の導入。
- **課金システムの導入**： 広告収入やユーザー課金を導入して利便性を図ります。

# 開発環境
- **Backend**: Ruby 3.2.0 / Ruby on Rails 7.1
- **Frontend**: React (Vite) / CSS-in-JS
- **判定ロジック**: 自作 RubyGem (`zombie_detector`)
- **Database**: PostgreSQL
- **Infrastructure**: AWS (EC2 / Ubuntu) / Cloudflare Tunnel (独自ドメイン)
- **Container**: Docker / Docker Compose
- **API**: X API v2 (Recent Search)

# 💪工夫したポイント
- **独自ロジックの Gem 化**: ゾンビ判定アルゴリズムを Rails 本体に記述するのではなく、`zombie_detector` という自作 Gem として分離しました。これにより、ロジックの再利用性とテストの容易性（疎結合）を高めました。また、ゾンビの流動的な性質変化に対応すべく、判定基準の分析を現在も行っております。
- **ゼロトラスト・ネットワーク構成**: AWS EC2 のポートを安易に開放せず、Cloudflare Tunnel を採用しました。外部からの直接アクセスを遮断しつつ、自費で取得してみた独自ドメインによる安全な HTTPS 通信を確立しています。
- **インフラのポータビリティ**: 開発環境と本番環境の差異をなくすため、Docker / Docker Compose を活用。ローカルで動作したコンテナをそのまま AWS 上で稼働させる設計にしました。

# ❤️‍🩹改善点
- **API通信時のテスト手法**: 現状は外部 API (X API) に依存する部分のテストが不十分なため、`WebMock` や `VCR` を導入し、外部通信をスタブ化（偽装）した堅牢な RSpec テストスイートを構築することで、アプリケーションの品質を担保したいと考えています。
